# Project Summarizer

Use this workflow after PRD, prototype, market, competitor, roadmap, or user research artifacts exist.

## Inputs

- Any generated PM Studio project artifacts.
- Target audience: executive, product team, engineering team, design team, investor, or customer.
- Desired format: one-page memo, meeting brief, release note, roadmap summary, or handoff.

## Output Sections

1. One-line product thesis.
2. Current decision needed.
3. Key user problem.
4. Proposed solution.
5. MVP scope.
6. Risks and assumptions.
7. Next actions.

## Process

1. Read only the artifacts relevant to the target audience.
2. Preserve decisions and open questions separately.
3. Remove duplicate details.
4. Highlight the smallest next action that moves the project forward.

## Quality Bar

- Summary must be shorter than the source by default.
- Do not bury decisions inside prose.
- Keep unresolved assumptions explicit.
- Avoid polishing away meaningful risk.
