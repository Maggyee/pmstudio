# Roadmap

## Phase 1: Agent Intake To Artifact Output

Goal: prove that one product idea can become a structured product pack.

- Define provider-neutral agent contract.
- Surface Codex and Claude Code as planned adapters.
- Map PM skills to concrete artifact outputs.
- Return mock product pack and PRD-to-prototype results from API routes.
- Show generated artifacts in the studio UI.

## Phase 2: PRD And Prototype Workspace

Goal: make text requirements and prototype screens reviewable together.

- Extract features, user tasks, IA, and page list from PRD.
- Generate prototype descriptions and preview pages.
- Add edit modes for PRD sections and prototype blocks.
- Add artifact export actions for Markdown, HTML, PDF, and PPTX.

## Phase 3: Multi-Agent Orchestration

Goal: coordinate specialized PM agents.

- Add workflow state machine for agent steps.
- Track agent events, outputs, and dependencies.
- Support reusable custom summary workflow.
- Add run history and project memory.

## Phase 4: OpenDesign-Style Runtime

Goal: connect real agent CLIs and sandbox generated prototypes.

- Add local CLI adapter layer for Codex and Claude Code.
- Add skill injection strategy.
- Add sandboxed HTML preview.
- Add generated artifact file storage.
- Add provider capability negotiation and graceful degradation.
